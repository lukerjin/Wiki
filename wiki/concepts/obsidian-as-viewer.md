# Obsidian as Viewer

> Using Obsidian as a rich, zero-config frontend for viewing an LLM-compiled knowledge base.

## Overview

Obsidian is an ideal viewer for an LLM-maintained wiki because it natively supports everything the system produces: markdown rendering, `[[wikilinks]]`, backlink graphs, Mermaid diagrams, and via plugins, Marp slides and other formats. The key insight is using Obsidian purely as a viewer — you don't write in it, the LLM writes for you.

## Key Points

- Obsidian renders markdown, wikilinks, and Mermaid diagrams natively
- The graph view provides automatic visualization of concept relationships
- Plugins extend rendering: Marp for slides, Dataview for queries
- Obsidian Web Clipper enables easy ingestion of web articles into `raw/`
- No configuration needed — just point Obsidian at the repo directory

## Recommended Plugins

- **Obsidian Web Clipper** — save web articles as .md into `raw/articles/`
- **Marp Slides** — render slide decks generated by the LLM
- **Dataview** — query and aggregate wiki data
- **Graph View** (built-in) — visualize concept connections
